## What this is

ML pipeline to predict **crystal strain and orientation** from **Kikuchi (EBSD) patterns** ‚Äî grayscale diffraction images from electron microscopy.

Each pattern is a **480√ó640 float32 image**. Labels are:
- `strain`: 6-component Voigt tensor `[Œµ11, Œµ22, Œµ33, Œµ23, Œµ13, Œµ12]`
- `orientation`: unit quaternion `[q0, q1, q2, q3]` (scalar-first convention)

Data is **fully synthetic**, generated by [EMsoft](https://github.com/EMsoft-org/EMsoft) via physics simulation.

## Architecture ‚Äî Two Phases

### Phase 1 (built, tested) ‚Äî Single pattern ‚Üí absolute strain

```
Input: (B, 1, H, W)   ‚Äî single grayscale Kikuchi pattern
  ‚Üí Encoder (ResNet18, 1-ch adapted, ImageNet pretrained) ‚Üí (B, feature_dim)
  ‚Üí StrainHead      (Linear ‚Üí ReLU ‚Üí Linear)  ‚Üí (B, 6)   Voigt strain
  ‚Üí OrientationHead (Linear ‚Üí ReLU ‚Üí Linear ‚Üí L2-norm) ‚Üí (B, 4)   unit quaternion  [optional]
```

Model file: `src/lbp_kikuchi/models/single_model.py`

### Phase 2 (built, not yet trained) ‚Äî Pairwise relative strain

Motivation: strain = small relative distortion between patterns, not absolute appearance. Mirrors HR-EBSD physics.

```
Grid A (3√ó3 neighbourhood around point A): (B, 9, 1, H, W)
Grid B (3√ó3 neighbourhood around point B): (B, 9, 1, H, W)
  ‚Üí Shared Encoder (same weights for all 18 patterns)
  ‚Üí grid A features: (B, feature_dim, 3, 3)
  ‚Üí grid B features: (B, feature_dim, 3, 3)
  ‚Üí relative features: F_B ‚àí F_A  ‚Üí (B, feature_dim, 3, 3)
  ‚Üí RelativeStrainHead (Conv2D ‚Üí Conv2D ‚Üí GAP ‚Üí FC ‚Üí FC) ‚Üí (B, 6)   ŒîŒµ
```

Output is **ŒîŒµ = Œµ_B ‚àí Œµ_A**, not absolute strain.  
Full strain field reconstructed by integration over grid (cumsum or least-squares).

Model file: `src/lbp_kikuchi/models/pair_model.py`

Grid ordering is **row-major** (index 4 = center pattern):
```
0 1 2
3 4 5
6 7 8
```

## Config ‚Äî `configs/encoder.yaml`

This is the **single source of truth** for all hyperparameters. Managed by [Hydra](https://hydra.cc/).

```yaml
experiment_name: "phase1_strain"
seed: 42

training:
  epochs: 50
  batch_size: 32
  lr: 1.0e-3
  val_split: 0.2
  norm_method: "minmax"      # "minmax" | "zscore"
  num_workers: 4
  predict_orientation: false # set true to attach OrientationHead
  orientation_loss_weight: 1.0

model:
  feature_dim: 128           # size of shared feature embedding from Encoder

data:
  path: "data/processed/"
  patterns_file: "X_patterns.npy"
  strain_file:   "y_strain.npy"
  orientation_file: "y_quaternion.npy"  # only loaded if predict_orientation=true
```

## Models

### `Encoder` ‚Äî `src/lbp_kikuchi/models/encoder.py`

```
Input:  (B, 1, H, W)   ‚Äî single-channel grayscale pattern
Output: (B, out_dim)   ‚Äî feature embedding vector
```

| Detail | Value |
|--------|-------|
| Base architecture | ResNet18 |
| Pretrained weights | `IMAGENET1K_V1` |
| Channel adaptation | `conv1` changed from 3‚Üí1 channel; weight initialised by averaging RGB weights (`w.mean(dim=1, keepdim=True)`) |
| Removed head | Original `fc` (512‚Üí1000) replaced with `nn.Linear(512, out_dim)` |
| Default `out_dim` | 128 (controlled by `model.feature_dim` in config) |
| Shared usage | Same weights used for all 18 patterns in Phase 2 via `PairModel._encode_grid` |

---

### `StrainHead` ‚Äî `src/lbp_kikuchi/models/heads.py`

```
Input:  (B, in_dim)   ‚Äî feature embedding
Output: (B, 6)        ‚Äî Voigt strain [Œµ11, Œµ22, Œµ33, Œµ23, Œµ13, Œµ12]
```

| Layer | Detail |
|-------|--------|
| Linear | `in_dim ‚Üí hidden_dim` (default `hidden_dim=64`) |
| ReLU | ‚Äî |
| Linear | `hidden_dim ‚Üí 6` |

No output activation ‚Äî raw regression values. The 6 outputs correspond to the **Voigt notation** ordering: `[Œµ11, Œµ22, Œµ33, Œµ23, Œµ13, Œµ12]`.

---

### `OrientationHead` ‚Äî `src/lbp_kikuchi/models/heads.py`

```
Input:  (B, in_dim)   ‚Äî feature embedding
Output: (B, 4)        ‚Äî unit quaternion [q0, q1, q2, q3]
```

| Layer | Detail |
|-------|--------|
| Linear | `in_dim ‚Üí hidden_dim` (default `hidden_dim=64`) |
| ReLU | ‚Äî |
| Linear | `hidden_dim ‚Üí 4` |
| L2-normalize | `F.normalize(q, p=2, dim=-1)` enforces unit norm |

The output lies on the unit 3-sphere **S¬≥**. Both `q` and `-q` represent the same physical rotation (double-cover). The loss and metrics account for this.

---

### `RelativeStrainHead` ‚Äî `src/lbp_kikuchi/models/heads.py`

```
Input:  (B, in_channels, 3, 3)   ‚Äî relative feature grid F_B ‚àí F_A
Output: (B, 6)                   ‚Äî predicted ŒîŒµ (Voigt format)
```

| Layer | Detail |
|-------|--------|
| Conv2d | `in_channels ‚Üí hidden_channels` (default 64), kernel 3√ó3, padding 1 |
| ReLU | ‚Äî |
| Conv2d | `hidden_channels ‚Üí hidden_channels`, kernel 3√ó3, padding 1 |
| ReLU | ‚Äî |
| Global Average Pool | `x.mean(dim=(-2, -1))` ‚Üí `(B, hidden_channels)` |
| Linear | `hidden_channels ‚Üí 32` |
| ReLU | ‚Äî |
| Linear | `32 ‚Üí 6` |

Uses Conv2D (not MLP) to explicitly model spatial neighbor structure ‚Äî analogous to digital image correlation (DIC).

### `PairModel` ‚Äî `src/lbp_kikuchi/models/pair_model.py`

**Phase 2 model.**

```
Input:
    grid_a: (B, 9, 1, H, W)   ‚Äî 9 patterns in 3√ó3 neighbourhood at point A
    grid_b: (B, 9, 1, H, W)   ‚Äî 9 patterns in 3√ó3 neighbourhood at point B

    'strain': (B, 6)           ‚Äî predicted ŒîŒµ = Œµ_B ‚àí Œµ_A (Voigt format)

```

| Attribute | Type | Description |
|-----------|------|-------------|
| `encoder` | `Encoder` | Shared weights for all 18 patterns |
| `head` | `RelativeStrainHead` | Conv2D spatial head |

```

  ‚Üí encoder: (B*9, feature_dim)
  ‚Üí reshape: (B, 9, feature_dim)
  ‚Üí permute: (B, feature_dim, 9)
  ‚Üí reshape: (B, feature_dim, 3, 3)
```

Extra method: `get_grid_features(grid)` ‚Üí `(B, feature_dim, 3, 3)` ‚Äî for analysis.

Constructor signature:
```python

```

|------|-------|-------|-------------|
| `X_patterns.npy` | `(N, H, W)` | float32 | Raw EBSD patterns |

## Datasets ‚Äî `src/lbp_kikuchi/data/dataset.py`

### Normalisation functions

```python
compute_norm_stats(X: np.ndarray, method: str) -> dict
```
- `method="minmax"` ‚Üí `{"min": float, "max": float}`
- `method="zscore"` ‚Üí `{"mean": float, "std": float}`

```python
apply_norm(X: np.ndarray, stats: dict, method: str) -> np.ndarray
```
- minmax: `(X - min) / (max - min + 1e-8)`
- zscore: `(X - mean) / (std + 1e-8)`

**Rule**: stats are **always computed on the training split only**. The same stats are passed to the validation split via the `stats` parameter. This prevents data leakage.

---

### `EBSDDataset`

Single-pattern dataset. One `(pattern_tensor, targets_dict)` per sample.

```python
EBSDDataset(
    X:           np.ndarray,          # (N, H, W) raw float32 patterns
    targets:     dict[str, np.ndarray],  # {'strain': (N,6), 'orientation': (N,4)}
    stats:       dict | None = None,  # None ‚Üí computed from X (train split only)
    norm_method: str = "minmax",
)
```

`__getitem__` returns:
```python
(
    torch.Tensor,   # (1, H, W)  float32 ‚Äî normalised pattern with channel dim added
    dict            # {'strain': Tensor(6,), ['orientation': Tensor(4,)]}
)
```

Stored attributes: `self.X` ‚Äî `(N, 1, H, W)` float32 tensor; `self.targets` ‚Äî dict of float32 tensors; `self.stats` ‚Äî normalisation stats; `self.norm_method`.

---

### `GridPairDataset`

Pairwise 3√ó3 grid dataset for Phase 2. One `(grid_a, grid_b, targets_dict)` per sample.

```python
GridPairDataset(
    grids_a:     np.ndarray,          # (N, 9, H, W) patterns at point A
    grids_b:     np.ndarray,          # (N, 9, H, W) patterns at point B
    targets:     dict[str, np.ndarray],  # {'strain': (N,6)} ‚Äî ŒîŒµ labels
    stats:       dict | None = None,  # None ‚Üí computed from both grids combined
    norm_method: str = "minmax",
)
```

`__getitem__` returns:
```python
(
    torch.Tensor,   # (9, 1, H, W)  float32 ‚Äî grid A
    torch.Tensor,   # (9, 1, H, W)  float32 ‚Äî grid B
    dict            # {'strain': Tensor(6,)}
)
```

When `stats=None`, normalisation stats are computed from **all patterns in both grids combined** (flattened to `(N*9*2, H, W)` before computing stats).

Grid index ‚Üí spatial position mapping:
```
index 0 1 2
      3 4 5   ‚Üê index 4 is the center pattern
      6 7 8
```

## Training ‚Äî `src/lbp_kikuchi/training/`

### Losses ‚Äî `losses.py`

#### `strain_loss(pred, target) ‚Üí Tensor`
- Plain `F.mse_loss` over all 6 Voigt components.
- `pred`, `target`: `(B, 6)` float32.

#### `quaternion_geodesic_loss(q_pred, q_true) ‚Üí Tensor`
- Geodesic (angular) distance on S¬≥.
- Handles q / ‚àíq double-cover: `dot = (q_pred * q_true).sum(dim=-1).abs().clamp(0,1)`
- Loss: `(2 * acos(dot)).mean()` ‚Äî result in **radians**.
- `q_pred`, `q_true`: `(B, 4)` unit quaternions.

#### `combined_loss(outputs, targets, orientation_loss_weight=1.0) ‚Üí Tensor`
- Dispatches to `strain_loss` always.
- Adds `orientation_loss_weight * quaternion_geodesic_loss(...)` if `'orientation'` key is present in both `outputs` and `targets`.
- Both args are `dict` with keys `'strain'` and optionally `'orientation'`.

### Metrics ‚Äî `metrics.py`

All metrics operate on full-epoch concatenated tensors (not batch-by-batch).

| Function | Input | Output | Description |
|----------|-------|--------|-------------|
| `strain_rmse(pred, target)` | `(N,6), (N,6)` | `float` | RMSE across all components |
| `strain_mae(pred, target)` | `(N,6), (N,6)` | `float` | MAE across all components |
| `strain_mae_per_component(pred, target)` | `(N,6), (N,6)` | `dict[str, float]` | Per-component MAE: keys `mae_e11` ‚Ä¶ `mae_e12` |
| `quaternion_angular_error_deg(q_pred, q_true)` | `(N,4), (N,4)` | `float` | Mean angular error in **degrees**, handles double-cover |
| `compute_all_metrics(outputs, targets)` | `dict, dict` | `dict[str, float]` | Aggregates all above into flat dict |

Voigt component names used as metric keys: `VOIGT_NAMES = ["e11", "e22", "e33", "e23", "e13", "e12"]`

`compute_all_metrics` output dict keys (when both strain and orientation present):
```
strain_rmse, strain_mae,
mae_e11, mae_e22, mae_e33, mae_e23, mae_e13, mae_e12,
orientation_error_deg
```

### Engine ‚Äî `engine.py`

#### `train_one_epoch(model, loader, optimizer, device, orientation_loss_weight=1.0) ‚Üí dict[str, float]`

- Sets `model.train()`.
- Loops over `(X, targets)` batches.
- Forward ‚Üí `combined_loss` ‚Üí `backward` ‚Üí `optimizer.step()`.
- Accumulates all predictions and targets.
- Returns `compute_all_metrics(...)` merged with `{"loss": mean_batch_loss}`.

#### `evaluate(model, loader, device, orientation_loss_weight=1.0) ‚Üí dict[str, float]`

- `@torch.no_grad()`, sets `model.eval()`.
- Same structure as `train_one_epoch`, no backward pass.
- Returns same metrics dict format.

Both functions use two internal helpers:
- `_to_device(targets: dict, device)` ‚Äî moves all tensors in a dict to device.
- `_cat_dicts(list_of_dicts)` ‚Äî concatenates matching keys across batch list using `torch.cat`.

## Scripts

### `scripts/train_encoder.py` ‚Äî Phase 1 training entrypoint

Decorated with `@hydra.main(config_path="../configs", config_name="encoder")`.

**Full training flow:**
1. `seed_everything(cfg.seed)`
2. Detect device (`cuda` if available, else `cpu`)
3. Create `run_dir = HydraConfig.get().runtime.output_dir` + `checkpoints/` subdirectory
4. Init `Logger`, call `logger.log_config(cfg_to_dict(cfg))`
5. Load `X_patterns.npy`, `y_strain.npy` (and `y_quaternion.npy` if `predict_orientation=true`)
6. Random train/val split: `val_size = int(val_split * N)`; shuffle with `torch.randperm(N)`
7. Compute `train_stats` from train indices only via `compute_norm_stats`
8. Build `EBSDDataset` for both splits (val receives `stats=train_stats`)
9. Build `DataLoader`s with `pin_memory=True`
10. Build `SinglePatternModel(feature_dim, predict_orientation)`
11. `Adam` optimizer, `CosineAnnealingLR(T_max=epochs)` scheduler
12. Epoch loop:
    - `train_one_epoch` ‚Üí `train_metrics`
    - `evaluate` ‚Üí `val_metrics`
    - Log merged dict `{epoch, lr, train_*, val_*}`
    - Save `last.pt`; save `best.pt` if val loss improved
    - `scheduler.step()`

**Outputs** (in `outputs/YYYY-MM-DD/HH-MM-SS/`):
```
checkpoints/best.pt          lowest val loss checkpoint
checkpoints/last.pt          most recent epoch checkpoint
metrics.json                 full per-epoch metrics history
```

## Key design decisions

| Decision | Rationale |
|----------|-----------|
| **Dict-based model outputs** | All models return `{'strain': ..., 'orientation': ...}`. Consistent across Phase 1 and Phase 2, extensible without breaking existing code. |
| **Dict-based dataset targets** | `EBSDDataset(X, {'strain': y, 'orientation': q})`. PyTorch's default collate handles dicts natively ‚Äî no custom collate needed. |
| **Normalisation from train split only** | `compute_norm_stats` called on train indices, same stats passed to val. Prevents leakage; val distribution is unknown at train time. |
| **Encoder shared in Phase 2** | `PairModel._encode_grid` calls `self.encoder` in a reshape loop ‚Äî same weights for all 18 patterns by construction. Ensures consistent feature space. |
| **Hydra output dir** | `HydraConfig.get().runtime.output_dir` (not `Path.cwd()` which gives repo root). Critical ‚Äî without this, all checkpoints land in the repo root. |
| **Quaternion loss handles double-cover** | Geodesic distance with `.abs()` before `acos` ensures q and ‚àíq give zero loss for the same rotation. |
| **Pattern normalisation at dataset level** | Done in `EBSDDataset.__init__` ‚Äî the DataLoader always yields already-normalised tensors. Avoids re-normalising at train time. |
| **ResNet18 grayscale adaptation** | Averaged RGB weights (`w.mean(dim=1)`) rather than random init ‚Äî preserves pretrained spatial filters for better transfer learning from natural images. |
| **CosineAnnealingLR** | Smooth LR decay from `lr` to ~0 over `epochs`. No warmup currently. |
| **`best.pt` vs `last.pt`** | `best.pt` = lowest val loss (use for inference/evaluation). `last.pt` = most recent (use for resuming). |
