# Project Context

## What this is

ML pipeline to predict **crystal strain and orientation** from **Kikuchi (EBSD) patterns** — grayscale diffraction images from electron microscopy.

Each pattern is a 480×640 float32 image. Labels are:
- `strain`: 6-component Voigt tensor `[ε11, ε22, ε33, ε23, ε13, ε12]`
- `orientation`: unit quaternion `[q0, q1, q2, q3]` (Bunge Euler convention)

Data is **fully synthetic**, generated by [EMsoft](https://github.com/EMsoft-org/EMsoft) via physics simulation.

---

## Architecture — Two Phases

### Phase 1 (built, tested) — Single pattern → absolute strain

```
(B, 1, H, W)
  → Encoder (ResNet18, 1-ch adapted, ImageNet pretrained) → (B, 128)
  → StrainHead (MLP)       → (B, 6)   Voigt strain
  → OrientationHead (MLP)  → (B, 4)   unit quaternion  [optional]
```

Model: `src/lbp_kikuchi/models/single_model.py`

### Phase 2 (built, not yet trained) — Pairwise relative strain

Motivation: strain = small relative distortion between patterns, not absolute appearance. Mirrors HR-EBSD physics.

```
Grid A (3×3 neighbourhood around point A): (B, 9, 1, H, W)
Grid B (3×3 neighbourhood around point B): (B, 9, 1, H, W)
  → Shared Encoder (same weights) → (B, C, 3, 3) per grid
  → F_B - F_A                     → (B, C, 3, 3)  relative features
  → RelativeStrainHead (Conv2D → GAP → FC) → (B, 6)  Δε
```

Output is **Δε = ε_B − ε_A**, not absolute strain.
Full strain field reconstructed by integration over grid (cumsum or least-squares).

Model: `src/lbp_kikuchi/models/pair_model.py`

Grid ordering is **row-major**:
```
0 1 2
3 4 5   (index 4 = center pattern)
6 7 8
```

## Data pipeline

```
scripts/generate_angles.py     →  data/raw/*.txt      (Euler + F tensors for EMsoft)
EMsoft (Docker)                →  data/raw/*.h5        (simulated patterns + labels)
scripts/prepare_numpy.py       →  data/processed/*.npy
scripts/train_encoder.py       →  outputs/DATE/TIME/
```

**Critical**: patterns are float32 in HDF5. Never convert to PNG for training — PNG is 8-bit and destroys the subtle intensity differences that encode strain. PNG is visualization only.

**Data must be a continuous strain field** (not random per-point samples) for Phase 2 to work — neighbouring points need physically consistent Δε labels.

## Running

```bash
# 1. Generate orientation/strain input for EMsoft
python scripts/generate_angles.py -n 10000 --strain-type multiaxial -s 0.02 --seed 42

# 2. Run EMsoft (inside Docker) → produces .h5 file

# 3. Convert to .npy
python scripts/prepare_numpy.py --h5 data/raw/Fe_EBSD_patterns.h5 --out data/processed/

# 4. Train Phase 1
python scripts/train_encoder.py

# Override config on CLI (Hydra):
python scripts/train_encoder.py training.lr=5e-4 model.feature_dim=256
python scripts/train_encoder.py training.predict_orientation=true
```

Outputs go to `outputs/YYYY-MM-DD/HH-MM-SS/` per run:
- `checkpoints/best.pt` — lowest val loss
- `checkpoints/last.pt` — latest epoch
- `metrics.csv` / `metrics.json` — per-epoch metrics
- `config_snapshot.json` — exact config used

## Key design decisions

- **Dict-based outputs**: all models return `{'strain': ..., 'orientation': ...}` — consistent across Phase 1 and Phase 2, extensible.
- **Dict-based dataset targets**: `EBSDDataset(X, {'strain': y, 'orientation': q})` — PyTorch's collate handles dicts natively.
- **Normalisation stats from train split only**: `compute_norm_stats` called on train indices, same stats applied to val. Prevents leakage.
- **Encoder shared in Phase 2**: `PairModel._encode_grid` calls `self.encoder` in a reshape loop — same weights for all 18 patterns by construction.
- **Hydra output dir**: `HydraConfig.get().runtime.output_dir` (not `Path.cwd()` which gives repo root).
- **Quaternion loss**: geodesic distance with `.abs()` before acos handles q/-q double-cover.

## Codebase structure

```
src/lbp_kikuchi/
  data/
    loader.py       HDF5 → numpy (EBSDDataLoader)
    dataset.py      numpy → PyTorch Dataset (EBSDDataset, GridPairDataset)
  models/
    encoder.py      ResNet18 adapted to grayscale
    heads.py        StrainHead, OrientationHead, RelativeStrainHead
    single_model.py Phase 1: encoder + heads
    pair_model.py   Phase 2: pairwise grid model
  training/
    losses.py       strain MSE, quaternion geodesic loss
    metrics.py      RMSE, per-component MAE, orientation angular error
    engine.py       train_one_epoch, evaluate
  utils/
    seed.py         seed_everything
    config.py       cfg_to_dict (Hydra → plain dict for logging)
    logger.py       CSV + JSON per-epoch logger

scripts/
  generate_angles.py   generate EMsoft input (orientations + F tensors)
  prepare_numpy.py     HDF5 → .npy conversion
  train_encoder.py     Phase 1 training entrypoint
  visualize.py         pattern visualisation (not part of training)

configs/
  encoder.yaml         all hyperparams and data paths
```
