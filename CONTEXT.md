# Project Context — lbp_kikuchi / pyEBSDtools

High-level map of the repository: synthetic EBSD data generation, three-stage strain ML, and evaluation. For day-to-day commands, see **NOTES.md**.

---

## What this is

ML pipeline to predict **crystal strain** (and optionally **orientation**) from **Kikuchi (EBSD) patterns** — grayscale diffraction images from electron microscopy.

Each pattern is a **480×640 float32 image**. Labels include:

- **`strain`**: 6-component Voigt tensor `[ε11, ε22, ε33, ε23, ε13, ε12]`
- **`orientation`**: unit quaternion `[q0, q1, q2, q3]` (scalar-first), when enabled

Data is **fully synthetic**, generated by [EMsoft](https://github.com/EMsoft-org/EMsoft) via physics simulation (Docker workflow in `datagen/`).

## Data pipeline

### Overview

```
datagen/configs/config.yaml
  │
  ├─ make sample   → datagen/sampler.py → datagen/angle_generation.py
  │                  writes angles, Ftensors, euler, optional y_positions, snapshot
  │
  ├─ make simulate → datagen/emsoft.py → Docker / EMsoft → Fe_EBSD_patterns.h5
  │
  ├─ make convert  → datagen/convert.py → helpers
  │                  writes X_patterns.npy, y_strain.npy, y_quaternion.npy, …
  │
  └─ make validate → helpers/validate.py
```

`make generate` runs the full chain. See `make help` for independent stages.

> **Critical**: patterns stay **float32** end-to-end. Do not round-trip through 8-bit PNG for training.

> **Stage 2 / 3**: labels should behave like a **spatially consistent strain field** (smooth or structured), not independent random noise per scan pixel, or neighbour-based models have nothing coherent to learn.

### Processed `.npy` shapes (typical)

| File | Shape | dtype | Description |
|------|-------|-------|-------------|
| `X_patterns.npy` | `(N, 1, H, W)` | float32 | Channel-first patterns |
| `y_strain.npy` | `(N, 6)` | float64 | Voigt strain |
| `y_quaternion.npy` | `(N, 4)` | float64 | Unit quaternion (if generated) |
| `y_euler.npy` | `(N, 3)` | float64 | Euler degrees (reference) |
| `y_positions.npy` | `(N, 2)` | int32 | `[row, col]` — explicit scan indices when used |

Default **H=480, W=640**. Flat index `i` ↔ `row = i // grid_cols`, `col = i % grid_cols` (row-major).

## Architecture — three ML stages

### Stage 1 — Single pattern → absolute strain (baseline)

```
(B, 1, H, W) → Encoder (ResNet18, 1-ch) → (B, D) → StrainHead → (B, 6)
                              → OrientationHead → (B, 4)   [optional]
```

- **Model**: `single_model.py`  
- **Train**: `scripts/train_encoder.py` · **Config**: `configs/encoder.yaml`  
- **Inference**: `scripts/infer.py` (Stage-1 runs only)

Ill-posed for tiny strains; useful as a sanity / encoder baseline.

### Stage 2 — 3×3 grid → ε at patch centre

```
(B, 9, 1, H, W) → shared encoder → (B, D, 3, 3) → SpatialStrainHead → (B, 6)
```

- **Model**: `grid_model.py`  
- **Train**: `scripts/train_grid.py` · **Config**: `configs/grid.yaml`  
- **Data**: interior scan centres with a full 3×3 neighbourhood (`build_grid_index`).

### Stage 3 — Two 3×3 grids → relative strain Δε

```
Grid A, Grid B → shared encoder each → difference map (B, D, 3, 3)
  → RelativeStrainHead → (B, 6)   Δε = ε_B − ε_A
```

- **Model**: `pair_model.py`  
- **Train**: `scripts/train_pair.py` · **Config**: `configs/pair.yaml`  
- **Pairs**: adjacent interior centres (`build_pair_index`; e.g. horizontal + vertical).

**Target normalisation**: Δε is normalised with the **same** `y_mean` / `y_std` as absolute strain (saved in `norm_stats.json`), matching training.

**Optional post-hoc ε field (evaluation only)**: `src/lbp_kikuchi/evaluation/spatial_plots.py` can integrate predicted Δε over **all** graph edges with `scipy.sparse.linalg.lsqr`, boundary ring pinned to ground-truth ε. This is **not** the training objective; it is a heavy, qualitative visualisation behind `infer_eval.py --full-spatial-viz`.

Voigt grid layout for the 9 patches:

```
0 1 2
3 4 5    ← index 4 is the centre pattern
6 7 8
```

## Datasets — `src/lbp_kikuchi/data/dataset.py`

### Normalisation

- Stats from **training split only** (`compute_norm_stats`); `apply_norm` for val/test.
- Stages 2–3: input stats use the **union** of all flat pattern indices that appear in any **training** 3×3 patch (so held-out pixels can lie outside [0,1] for min–max).

### Class overview

| Class | Used by | Role |
|--------|---------|------|
| `EBSDDataset` | Stage 1 training | One pattern → strain (+ optional quaternion) |
| `GridDataset` / `GridPairDataset` | Legacy / tests | Eager stacked grids (memory-heavy on large N) |
| **`LazyGridDataset`** | `train_grid.py`, `infer_eval` Stage 2 | Builds each 3×3 patch in `__getitem__` from flat `X` |
| **`LazyGridPairDataset`** | `train_pair.py`, `infer_eval` Stage 3 | Two patches + Δε target + `pos_a`, `pos_b` `[row,col]` |

Pair items return **`(grid_a, grid_b, targets_dict, pos_a, pos_b)`** for Saint–Venant and for eval scatter positions.

## Training — `src/lbp_kikuchi/training/`

### Losses (`losses.py`)

- **Strain**: Huber / MAE / MSE on Voigt components.
- **Quaternion**: geodesic on S³; double-cover handled in loss.
- **`physical_bounds_loss`**: soft penalty if |Δε| exceeds a configured cap.
- **`loop_consistency_loss`**: discrete **Saint–Venant**-style closed-loop residual on predicted Δε over rectangular cells when batch positions form loops; **0** if no loops in the batch.
- **`pair_loss`**: `L_reg + sv_weight * L_sv + bounds_weight * L_bounds` with per-term dict for logging.

### Engine (`engine.py`)

- `train_one_epoch` / `evaluate` return merged loss + `compute_all_metrics(...)`.
- Training uses **tqdm** on batches (also used in `infer_eval` for optional full-scan pass).

### Run artefacts (`outputs/YYYY-MM-DD/HH-MM-SS/`)

- `checkpoints/best.pt`, `last.pt`, `norm_stats.json`
- `config_snapshot.json`, `metrics.csv` / `metrics.json`

## Evaluation — `scripts/infer_eval.py`

Single entrypoint: reads `config_snapshot.json`, detects **Stage 1 / 2 / 3** from the config, loads the right model, runs the chosen **split** (`train` / `val` / `test` / `all`), writes **`eval_results_<split>.json`** and parity **`eval_scatter_<split>.png`** (physical units).

### Normalised vs physical metrics

- **TensorBoard** `strain_mae` / `strain_rmse`: on **z-scored** targets (`pred` in normalised space).
- **`infer_eval`**: converts with **`pred_phys = pred_norm * y_std + y_mean`** before MAE/RMSE and plot axes — **scales differ**; trends should still correlate.

### CLI (main flags)

| Flag | Effect |
|------|--------|
| `--run-dir PATH` | Run to evaluate (default: latest under `outputs/`) |
| `--split {val,train,test,all}` | Which index split (default `val`) |
| `--checkpoint best.pt` | Checkpoint name under `checkpoints/` |
| `--batch-size N` | Inference batch size |
| `--max-samples N` | Cap rows for metrics / scatter / Stage-3 Δε-vs-xy |
| `--no-plot` | Skip all figures |
| `--no-spatial-viz` | Skip optional spatial outputs (including Stage-3 Δε vs row/col) |
| **`--full-spatial-viz`** | **Second** full-dataset forward pass + heatmaps / optional LSQR ε (Stages 1–3); **off by default** |

### Default spatial output (Stage 3 only, when plots on)

- **`eval_spatial_stage3_delta_vs_xy_<split>.png`**: six rows × two columns — **true Δε** and **predicted Δε** as scatter vs **(col, row)** at edge midpoints, using **only the eval split** — **no extra inference**.

### Optional heavy spatial (`--full-spatial-viz`)

- Stage 1 / 2: full-scan ε heatmaps (true | pred | |error|).
- Stage 3: infer **all** pairs again, LSQR reconstruct ε, dense midpoint Δε PNGs. **Expensive**; meant for exploration, not required for parity metrics.

Implementation: **`src/lbp_kikuchi/evaluation/spatial_plots.py`** (uses **SciPy** sparse LSQR when that path runs).

## Running

```bash
# ── Data generation ──────────────────────────────────────────────────────────
make generate
make sample / make simulate / make convert / make validate   # see Makefile

# ── Training (Hydra overrides use key=value, no leading --) ────────────────
python scripts/train_encoder.py data.path=data/processed/
python scripts/train_grid.py    data.path=data/processed/ data.grid_rows=100 data.grid_cols=100
python scripts/train_pair.py    data.path=data/processed/ data.grid_rows=100 data.grid_cols=100

# ── Evaluation (all stages, auto-detect) ─────────────────────────────────────
python scripts/infer_eval.py --split test
python scripts/infer_eval.py --run-dir outputs/YYYY-MM-DD/HH-MM-SS --no-spatial-viz
python scripts/infer_eval.py --full-spatial-viz   # optional second pass + maps

# ── Single-sample demo (Stage 1 checkpoint only) ───────────────────────────
python scripts/infer.py --run-dir outputs/... --index 0
```

Data generation also writes **`config_snapshot.yaml`** next to the experiment data under your configured EMsoft data root.

## Key design decisions

| Topic | Choice |
|--------|--------|
| Model I/O | Dicts: `{'strain': ..., 'orientation': ...}` |
| Targets | Dicts in datasets; default PyTorch collate |
| Encoder Stage 2/3 | Same weights: reshape `(B*9, 1, H, W)` through encoder |
| Large scans | **Lazy** grid datasets: read `X_patterns.npy` once, build 3×3 in `__getitem__` |
| Hydra cwd | Use `HydraConfig.get().runtime.output_dir` for logs/checkpoints |
| Stage 3 positions | `pos_a` / `pos_b` as `[row, col]` for SV loss and eval plots |
| Eval stage routing | `infer_eval.py` inspects config for pair vs grid vs single |
