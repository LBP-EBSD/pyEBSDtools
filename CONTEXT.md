# Project Context — lbp_kikuchi / EBSDtools

---

## What this is

ML pipeline to predict **crystal strain and orientation** from **Kikuchi (EBSD) patterns** — grayscale diffraction images from electron microscopy.

Each pattern is a **480×640 float32 image**. Labels are:
- `strain`: 6-component Voigt tensor `[ε11, ε22, ε33, ε23, ε13, ε12]`
- `orientation`: unit quaternion `[q0, q1, q2, q3]` (scalar-first convention)

Data is **fully synthetic**, generated by [EMsoft](https://github.com/EMsoft-org/EMsoft) via physics simulation.

## Data pipeline

### Overview

```
datagen/configs/config.yaml
  │
  ├─ make sample   → datagen/sampler.py → datagen/angle_generation.py
  │                  writes: {exp}_angles.txt  {exp}_Ftensors.npy  {exp}_euler.npy
  │                  also saves: config_snapshot.yaml alongside the data
  │
  ├─ make simulate → datagen/emsoft.py  → Docker: EMMCOpenCL → EMEBSDmaster → EMEBSD
  │                  writes: Fe_EBSD_patterns.h5
  │
  ├─ make convert  → datagen/convert.py → helpers/{hdf5_io, crystal}
  │                  writes: X_patterns.npy (N,1,H,W)  y_strain.npy  y_quaternion.npy  y_euler.npy
  │
  └─ make validate → helpers/validate.py
```

`make generate` runs all four stages in order. All stages can be skipped independently — see `make help`.

> **Critical**: patterns are float32 in HDF5. Never convert to PNG for training — PNG is 8-bit and destroys the subtle intensity differences that encode strain. PNG is for visualisation only.

> **Data must be a continuous strain field** (not random per-point samples) for ML Stage 2/3 to work — neighbouring scan points need physically consistent Δε labels.

### Processed `.npy` file shapes

| File | Shape | dtype | Description |
|------|-------|-------|-------------|
| `X_patterns.npy` | `(N, 1, H, W)` | float32 | Raw EBSD patterns, channel-first |
| `y_strain.npy` | `(N, 6)` | float64 | Voigt strain labels |
| `y_quaternion.npy` | `(N, 4)` | float64 | Unit quaternion labels |
| `y_euler.npy` | `(N, 3)` | float64 | Euler angles in degrees (reference only) |
| `y_positions.npy` | `(N, 2)` | int32 | `[row, col]` scan-grid position — **spatial mode only** |

Default pattern size: **H=480, W=640**.

> **Position ordering**: pattern `i` lives at `row = i // grid_cols`, `col = i % grid_cols` (row-major C order, same as EMsoft scan order). `y_positions.npy` makes this explicit and is required by the Saint-Venant loop-consistency loss in Stage 3.

## Architecture — Three ML Stages

### ML Stage 1 (built, tested) — Single pattern → absolute strain

```
(B, 1, H, W)
  → Encoder (ResNet18, 1-ch adapted, ImageNet pretrained) → (B, feature_dim)
  → StrainHead (MLP)       → (B, 6)   Voigt strain
  → OrientationHead (MLP)  → (B, 4)   unit quaternion  [optional]
```

Model: `src/lbp_kikuchi/models/single_model.py`
Train: `scripts/train_encoder.py`  Config: `configs/encoder.yaml`

Purpose: debug baseline only — mapping is ill-posed (single pattern is ambiguous
for small strain) and the model collapses to predicting near-zero strain.

---

### ML Stage 2 (built) — 3×3 grid → absolute strain at center

```
Grid (3×3 neighbourhood): (B, 9, 1, H, W)
  → Shared Encoder (same weights for all 9 patterns) → (B, feature_dim, 3, 3)
  → SpatialStrainHead (Conv2D → GAP + center-skip → FC) → (B, 6)  ε(center)
```

Spatial context from neighbours partially breaks the ambiguity of Stage 1:
the model can now learn spatial gradients (how strain varies across the scan).

Model: `src/lbp_kikuchi/models/grid_model.py`
Train: `scripts/train_grid.py`  Config: `configs/grid.yaml`

---

### ML Stage 3 (built) — Pair of 3×3 grids → relative strain Δε

Motivation: strain = small relative distortion between patterns, not absolute appearance. Mirrors HR-EBSD physics.

```
Grid A (3×3 neighbourhood around point A): (B, 9, 1, H, W)
Grid B (3×3 neighbourhood around point B): (B, 9, 1, H, W)
  → Shared Encoder (same weights) → (B, feature_dim, 3, 3) per grid
  → F_B - F_A                     → (B, feature_dim, 3, 3)  relative features
  → RelativeStrainHead (Conv2D → GAP → FC) → (B, 6)  Δε
```

Output is **Δε = ε_B − ε_A**, not absolute strain.
Full strain field reconstructed by integration over grid (cumsum or least-squares).

Model: `src/lbp_kikuchi/models/pair_model.py`
Train: `scripts/train_pair.py`  Config: `configs/pair.yaml`

Grid ordering is **row-major**:
```
0 1 2
3 4 5   (index 4 = center pattern)
6 7 8
```

## Running

```bash
# ── Data generation ──────────────────────────────────────────────────────────
# Edit datagen/configs/config.yaml, then:
make generate                        # full pipeline (stages 1–4)
make generate CONFIG=datagen/configs/my_experiment.yaml   # custom config
make sample                          # Stage 1 only: angles + labels
make simulate                        # Stage 2 only: Docker/EMsoft → .h5
make convert                         # Stage 3 only: .h5 → .npy
make validate                        # Stage 4 only: sanity check
make preview                         # visualise generated patterns

# ── Training ─────────────────────────────────────────────────────────────────
python scripts/train_encoder.py data.path=data/processed/
python scripts/train_grid.py    data.path=data/processed/ data.grid_rows=100 data.grid_cols=100
python scripts/train_pair.py    data.path=data/processed/ data.grid_rows=100 data.grid_cols=100

# Override any config on CLI (Hydra syntax — no -- prefix):
python scripts/train_grid.py training.lr=5e-4 model.feature_dim=256
python scripts/train_pair.py training.batch_size=4 training.epochs=100
```

Training outputs go to `outputs/YYYY-MM-DD/HH-MM-SS/` per run:
- `checkpoints/best.pt` — lowest val loss
- `checkpoints/last.pt` — latest epoch
- `metrics.csv` / `metrics.json` — per-epoch metrics
- `config_snapshot.json` — exact config used

Data generation outputs a `config_snapshot.yaml` alongside every experiment's data:
```
~/EMsoftData/{experiment_name}/config_snapshot.yaml   ← exact config used for that run
```

## Models

### `Encoder` — `src/lbp_kikuchi/models/encoder.py`

| Detail | Value |
|--------|-------|
| Base architecture | ResNet18 |
| Pretrained weights | `IMAGENET1K_V1` |
| Channel adaptation | `conv1` changed from 3→1 channel; weight initialised by averaging RGB weights |
| Removed head | Original `fc` (512→1000) replaced with `nn.Linear(512, out_dim)` |
| Default `out_dim` | 128 (controlled by `model.feature_dim` in config) |
| Shared usage | Same weights used for all 9/18 patterns in Stage 2/3 |

---

### `StrainHead` — `src/lbp_kikuchi/models/heads.py`

`(B, in_dim) → Linear → ReLU → Linear → (B, 6)` Voigt strain. No output activation.

---

### `OrientationHead` — `src/lbp_kikuchi/models/heads.py`

`(B, in_dim) → Linear → ReLU → Linear → L2-normalize → (B, 4)` unit quaternion.
Output lies on S³. Both `q` and `−q` represent the same rotation — loss accounts for this.

---

### `RelativeStrainHead` — `src/lbp_kikuchi/models/heads.py`

`(B, C, 3, 3) → Conv2d → ReLU → Conv2d → ReLU → GAP → Linear → ReLU → Linear → (B, 6)` Δε.
Conv2D (not MLP) to explicitly model spatial neighbour structure — analogous to digital image correlation (DIC).

## Datasets — `src/lbp_kikuchi/data/dataset.py`

### Normalisation

- `compute_norm_stats(X, method)` — `"minmax"` or `"zscore"`, computed on **train split only**.
- `apply_norm(X, stats, method)` — applies the pre-computed stats.
- Stats are always computed from training split; same stats applied to val (no leakage).

### `EBSDDataset`

Single-pattern dataset for Stage 1. Returns `(1, H, W)` tensor + `{'strain': (6,), 'orientation': (4,)}`.

### `GridDataset`

3×3 grid dataset for Stage 2. Interior points only (boundary rows/cols discarded for complete 3×3 ring).
Returns `(9, 1, H, W)` grid tensor + `{'strain': (6,)}`.

### `GridPairDataset`

Pairwise dataset for Stage 3. Returns `(9, 1, H, W)` grid_a, `(9, 1, H, W)` grid_b + `{'strain': (6,)}` Δε + `pos_a (2,)` + `pos_b (2,)` scan positions.
Normalisation stats computed from all patterns in both grids combined.
Positions are `[row, col]` int32 tensors used by the Saint-Venant loop-consistency loss.

## Training — `src/lbp_kikuchi/training/`

### Losses — `losses.py`

- `strain_loss` — Huber / MAE / MSE over all 6 Voigt components.
- `quaternion_geodesic_loss` — geodesic distance on S³; handles q/−q double-cover via `.abs()` before `acos`.
- `combined_loss(outputs, targets)` — dispatches to both; orientation term scaled by `orientation_loss_weight`.
- `physical_bounds_loss(delta_pred, max_abs_strain)` — penalises Δε components that exceed a physical magnitude threshold (soft constraint, quadratic beyond threshold).
- `loop_consistency_loss(delta_pred, pos_a, pos_b)` — **Saint-Venant integrability**: finds all complete rectangular loops in the batch (using scan-grid positions) and penalises non-zero closed-loop residuals of predicted Δε. Zero when no loops are found or positions are unknown.
- `pair_loss(delta_pred, delta_target, pos_a, pos_b, ...)` — full Stage-3 loss: `L_reg + sv_weight * L_sv + bounds_weight * L_bounds`. Returns `(total_loss, components_dict)` for per-term TensorBoard logging.

### Metrics — `metrics.py`

| Function | Output |
|----------|--------|
| `strain_rmse` | scalar RMSE across all 6 components |
| `strain_mae` | scalar MAE |
| `strain_mae_per_component` | dict `{mae_e11, …, mae_e12}` |
| `quaternion_angular_error_deg` | mean angular error in degrees, handles double-cover |
| `compute_all_metrics` | aggregates all into one flat dict |

### Engine — `engine.py`

- `train_one_epoch(model, loader, optimizer, device)` → metrics dict
- `evaluate(model, loader, device)` → metrics dict

Both return `compute_all_metrics(...)` merged with `{"loss": mean_batch_loss}`.

## Key design decisions

| Decision | Rationale |
|----------|-----------|
| **Dict-based model outputs** | All models return `{'strain': ..., 'orientation': ...}`. Consistent across all stages. |
| **Dict-based dataset targets** | PyTorch's default collate handles dicts natively — no custom collate needed. |
| **Normalisation from train split only** | `compute_norm_stats` on train indices, same stats to val. Prevents leakage. |
| **Encoder shared in Stage 2/3** | `_encode_grid` reshapes `(B*9, 1, H, W)` → encoder → `(B, feature_dim, 3, 3)` — same weights for all patterns. |
| **Hydra output dir** | `HydraConfig.get().runtime.output_dir` (not `Path.cwd()` which gives repo root). |
| **Quaternion loss handles double-cover** | `.abs()` before `acos` ensures q and −q give zero loss for the same rotation. |
| **Pattern normalisation at dataset level** | Done in `__init__` — DataLoader always yields normalised tensors. |
| **ResNet18 grayscale adaptation** | Averaged RGB weights preserve pretrained spatial filters. |
| **`X_patterns.npy` is (N, 1, H, W)** | Channel-first from `datagen/convert.py`; ready for PyTorch without extra unsqueeze. |
| **Config snapshot saved with data** | `datagen/pipeline.py` writes `config_snapshot.yaml` to experiment dir — exact params that produced that `.h5`. |

## Key design decisions (additions)

| Decision | Rationale |
|----------|-----------|
| **`X_patterns.npy` is (N,1,H,W); build functions accept both** | `_squeeze_channel()` in `dataset.py` transparently strips the channel dim before building grids — avoids silent shape bugs. |
| **`y_positions.npy` saved for spatial data** | Row-major position is implicit in flat-index ordering, but saving it explicitly allows position-aware train/val splits and the SV loop loss without passing `grid_rows`/`grid_cols` everywhere. |
| **`GridPairDataset` returns 5-tuple** | `(grid_a, grid_b, targets, pos_a, pos_b)` — positions are -1 when unavailable (fallback); the SV loss silently returns 0.0 in that case. |
| **Saint-Venant loop-consistency loss** | Enforces integrability of predicted Δε: closed-loop residual on rectangular cells of the scan grid must be zero. Equivalent to discrete compatibility (curl-free gradient). Found from batch positions at training time with no extra data loading. |
| **Physical-bounds loss** | Soft quadratic penalty beyond `max_abs_strain` (default 5%) per component. Prevents the network from exploiting large spurious predictions to minimise regression loss. |
| **`pair_loss` returns `(loss, components_dict)`** | Allows per-term logging (`loss_reg`, `loss_sv`, `loss_bounds`) in TensorBoard without extra forward passes. |
